  <action>
    Create `Sources/UnibrainCore/Classification/CourseMapping.swift` containing three Codable, Sendable structs:

    1. `TermDefinition` — `var label: String`, `var startDate: Date`, `var endDate: Date`. CodingKeys: none needed (snake_case not required for leaf fields unless they contain acronyms — use camelCase to match Swift conventions; the document-level CodingKeys handle the outer key mapping).

    2. `CourseMapping` — `var courseCode: String`, `var courseName: String`.

    3. `CourseMappingDocument` — `var schemaVersion: Int` (CodingKey: "schema_version"), `var currentTerm: TermDefinition` (CodingKey: "current_term"), `var mappings: [String: CourseMapping]` (event title -> mapping), `var recentCourseCodes: [String]` (CodingKey: "recent_course_codes"). All vars are `var` so the store can mutate in place. Add a static factory `CourseMappingDocument.empty` that returns a document with schema_version=1, empty mappings, empty recent list, and a default term (label: "", startDate: Date.distantPast, endDate: Date.distantFuture — the wide default term ensures recordings are never accidentally filtered out before the user sets a real term).

    4. Use `JSONDecoder` with `dateDecodingStrategy: .iso8601` and `JSONEncoder` with `dateEncodingStrategy: .iso8601` for consistent date handling across iCloud-synced devices.

    Create `Sources/UnibrainCore/Classification/CourseMappingStore.swift` as a `public actor CourseMappingStore`:

    - Init takes `vaultRoot: URL`. Compute `storeURL` as `vaultRoot.appendingPathComponent(".unibrain").appendingPathComponent("courses.json")`.
    - `load() throws -> CourseMappingDocument`: Read storeURL via FileManager.default.contents(atPath:). If file missing or decode fails (malformed JSON), return `.empty` (never throws for load — per RESEARCH.md Pitfall: "malformed JSON falls back to empty default store, doesn't crash"). Log a warning via os.Logger if available (guard with `#if canImport(os)` to keep Linux-testable). On success, decode and return.
    - `save(_ document: CourseMappingStore_Document) throws`: Ensure `.unibrain/` directory exists via FileManager.createDirectory(withIntermediateDirectories: true). Encode with JSONEncoder. Write atomically via Data.write(to:options:.atomic) to prevent corruption from concurrent iCloud sync (DISC-06).
    - `lookup(eventTitle: String) throws -> CourseMapping?`: Load document, return `document.mappings[eventTitle]`.
    - `upsert(eventTitle: String, mapping: CourseMapping) throws`: Load, set `document.mappings[eventTitle] = mapping`, save. This is the M-02 auto-learn path and the M-03 manual-pick mapping update path.
    - `addRecent(courseCode: String) throws`: Load, remove any prior occurrence of courseCode from recentCourseCodes, insert at index 0, trim to first 5 elements, save. This is the M-03 recent list update.
    - `setCurrentTerm(label: String, startDate: Date, endDate: Date) throws`: Load, set document.currentTerm, save. This is CT-01.
    - `allMappings() throws -> [String: CourseMapping]`: Load, return mappings dict (for Manage Courses sheet).
    - `allRecentCourses() throws -> [String]`: Load, return recentCourseCodes array (for picker Recent section).
    - `currentTerm() throws -> TermDefinition`: Load, return currentTerm (for term filter and UI display).
    - `deleteMapping(eventTitle: String) throws`: Load, remove key from mappings, save (for Manage Courses delete action).

    All methods are `throws` because they perform file I/O. The actor isolation ensures serialized access — no concurrent writes (per Swift 6 actor model, matching PipelineOrchestrator and ModelLoadGate patterns from Phase 1).

    Per T-2-01 (path traversal): the `.unibrain/` path component is a fixed constant, not derived from user input. Event titles are used as dictionary keys in the JSON, not as path components — FolderNameSanitizer is applied downstream when creating folders, not in this store.
