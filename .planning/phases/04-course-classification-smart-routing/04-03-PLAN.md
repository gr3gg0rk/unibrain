    - Test 3: recentCourses returns at most 5 items from recentCodes, in order.

    - Test 9: createNew(code:name:) produces a CourseSelection.newCourse(code, name) result.
  </behavior>
  <action>
    Create `Sources/UnibrainCore/Classification/CoursePickerViewModel.swift`:

    Define `public enum CoursePickerMode: Sendable, Equatable`:
    - `.none` — zero events matched, show course list only (UI-SPEC Variant B)
    - `.multiple([CalendarEvent])` — multiple events matched, show events + course list (UI-SPEC Variant A)

    Define `public enum CourseSelection: Sendable`:
    - `.course(String)` — user picked an existing course code from the list
    - `.event(CalendarEvent)` — user picked a specific calendar event from multi-match list
    - `.newCourse(code: String, name: String)` — user created a new course via Create New form
    - `.skip` — user chose Skip, routing to _unsorted/

    Define `public struct CourseSummary: Sendable, Equatable, Identifiable`:
    - `public let code: String`
    - `public let name: String`
    - `public var id: String { code }`

    Define `public class CoursePickerViewModel` (plain class, NOT @Observable — that conformance is added in the UnibrainApp layer with a SwiftUI wrapper):
    - Properties: `public private(set) var mode: CoursePickerMode`, `public private(set) var allCourses: [CourseSummary]`, `public private(set) var recentCodes: [String]`, `public private(set) var selection: CourseSelection?`, `public var searchQuery: String = ""`
    - Init: `public init(mode: CoursePickerMode, courses: [CourseSummary], recentCodes: [String])`
    - Computed: `public var recentCourses: [CourseSummary]` — filters allCourses to those whose code is in recentCodes, ordered by recentCodes order, max 5 (MP-02).
    - Computed: `public var filteredCourses: [CourseSummary]` — when searchQuery is empty, returns allCourses; when non-empty, returns allCourses where code or name contains searchQuery (case-insensitive).
    - Computed: `public var matchingEvents: [CalendarEvent]?` — returns events from mode if `.multiple`, nil otherwise.
    - Method: `public func select(course: CourseSummary)` — sets selection to `.course(course.code)`.
    - Method: `public func selectEvent(_ event: CalendarEvent)` — sets selection to `.event(event)` (MP-05).
    - Method: `public func skip()` — sets selection to `.skip` (MP-03).
    - Method: `public func createNew(code: String, name: String)` — sets selection to `.newCourse(code: code, name: name)` (MP-03 Create New).

    This class is pure logic with no async operations — the actual CourseMappingStore.upsert/addRecent calls happen in the orchestrator/UI layer after the selection is made. The view model produces the selection; the consumer (orchestrator or view model coordinator) applies it. This keeps the class Linux-testable with no actor dependencies.

    The searchQuery property is a `var` (not `private(set)`) because the UI binds a TextField directly to it — the UI layer (UnibrainApp) wraps this in a SwiftUI @Observable adapter if needed, or uses it as-is with manual binding.
