## Phase Goal

**As a** new user, **I want to** walk through a guided first-run setup that picks my vault folder, grants permissions, and sets my current term, **so that** the app is ready to record without any manual configuration.

## Acceptance Criteria

- [ ] On first launch (hasCompletedOnboarding==false), the app shows OnboardingFlow instead of the main UI (ONBD-01)
- [ ] The vault picker page uses .fileImporter with UTType.folder and suggests iCloud Drive as the default location (ONBD-04)
- [ ] The picked vault folder URL persists across launches via security-scoped bookmark stored in Keychain (ONBD-04)
- [ ] The mic permission page blocks advancement if mic is denied (ONBD-02 hard-fail carried forward)
- [ ] The calendar permission page allows Skip (ONBD-03 optional carried forward)
- [ ] macOS onboarding includes a Term page (label + start/end dates); iOS skips it (ONB-01)
- [ ] The Ready page sets hasCompletedOnboarding=true and dismisses onboarding (ONBD-01)
- [ ] A Permissions sheet accessible post-onboarding shows live mic/calendar/vault status with Settings deep-links (ONBD-05)

    Create BookmarkStore in Sources/UnibrainProviders/Security/BookmarkStore.swift. Per ONB-03 and Pattern 5 from RESEARCH.md: a final class (Sendable via @unchecked) with three static methods: save(for: URL) throws, resolve() -> URL?, clear(). Uses Security framework (SecItemAdd/SecItemCopyMatching/SecItemDelete) with the key "app.unibrain.vault_bookmark". The save method calls url.bookmark(options: .withSecurityScope) and stores the resulting Data in Keychain as a generic password item. The resolve method loads the Data, calls URL(resolvingBookmarkData:options:relativeTo:bookmarkDataIsStale:) and checks bookmarkDataIsStale — if stale, returns nil (caller must re-prompt). After resolving, calls url.startAccessingSecurityScopedResource() before returning the URL. Guard all Keychain calls with #if os(macOS) || os(iOS) since SecItem APIs are unavailable on Linux. Per IC-01: this stores per-device in Keychain, NOT UserDefaults (security best practice).

Methods: pickVault(url: URL) -> saves bookmark via BookmarkStore.save; requestMicPermission() async -> uses AVAudioApplication.requestRecordPermission on macOS / AVAudioSession on iOS; requestCalendarPermission() async -> reuses EventKitCalendarAdapter.requestFullAccess(); saveTerm() async -> calls CourseMappingStore.setCurrentTerm; completeOnboarding() -> sets UserDefaults.standard.set(true, forKey: "hasCompletedOnboarding"). <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User-selected folder -> App filesystem access | Security-scoped bookmark grants the app read-write access to a user-picked folder; access must be scoped and revocable |
| Onboarding permission grants -> System privacy | Microphone and calendar permission requests cross into system privacy controls |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-01 | Information Disclosure | BookmarkStore | medium | mitigate | Store bookmark data in Keychain (SecItemAdd with kSecAttrAccessibleWhenUnlocked), never UserDefaults — bookmarks grant filesystem access |
| T-05-02 | Tampering | OnboardingVaultPage fileImporter | low | accept | .fileImporter returns system-validated URLs; no user-supplied string paths accepted |
| T-05-03 | Elevation of Privilege | BookmarkStore.resolve() | medium | mitigate | Check bookmarkDataIsStale flag; return nil if stale so caller re-prompts; call startAccessingSecurityScopedResource before returning URL |
| T-05-04 | Information Disclosure | PermissionsSheet status display | low | accept | Status shows only granted/denied boolean — no sensitive calendar or audio data exposed |
| T-05-SC | Tampering | No package installs in this plan | high | mitigate | N/A — zero external packages installed; all Apple built-in frameworks |
</threat_model>
