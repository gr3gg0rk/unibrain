## Phase Goal

**As a** student in a lecture hall, **I want to** record on my iPhone with the screen locked and have it survive interruptions, **so that** I can discreetly capture the lecture without my phone dying or the recording stopping.

## Acceptance Criteria

- [ ] User taps Record on iPhone, locks screen, and recording continues for 30+ minutes (CAPT-03, DISC-04)
- [ ] iOS lock screen displays "Recording — {elapsed}" with Stop/Pause buttons (IOS-02)
- [ ] A phone call or Siri interruption auto-pauses recording; auto-resumes when interruption ends (IOS-03)
- [ ] On Stop, the .m4a file moves from sandbox tmp/ to {vault}/_inbox/{source}-{timestamp}-{shortUUID}.m4a (IC-02)
- [ ] iOS app shell is a three-tab TabView: Record, Recent, Settings (IOS-01)
- [ ] Record tab shows full-screen timer (48pt monospaced), waveform, mic meter, Pause/Stop buttons (IOS-04)

Per Pitfall 4: the audio session must be active BEFORE setting nowPlayingInfo.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| iOS background audio session -> System audio mixing | The app claims the audio session in the background; must handle interruptions correctly |
| Sandbox tmp/ -> iCloud Drive _inbox/ | Audio file moves from app sandbox to user-picked iCloud folder; must be atomic |
| Lock screen remote commands -> Recording lifecycle | MPRemoteCommandCenter handlers control recording from outside the app UI |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-05 | Denial of Service | iOSAudioSessionManager | high | mitigate | Observe interruptionNotification and mediaServicesWereResetNotification; auto-pause on interruption prevents recording loss; auto-resume when interruption ends |
| T-05-06 | Information Disclosure | NowPlayingManager lock screen | medium | mitigate | Lock screen shows only "Recording — {elapsed}" — no course names, no transcript fragments, no audio content; just elapsed time |
| T-05-07 | Tampering | IC-03 filename generation | low | mitigate | Timestamp from ISO 8601 + 4-char UUID hex suffix prevents collision; FolderNameSanitizer (Phase 4) not needed for filenames but path components validated |
| T-05-08 | Repudiation | Recording start/stop audit trail | low | accept | Single-user app; recording lifecycle visible in UI and file timestamps; no multi-party audit needed |
| T-05-SC | Tampering | No package installs in this plan | high | mitigate | N/A — zero external packages; all Apple built-in frameworks (AVFoundation, MediaPlayer) |
