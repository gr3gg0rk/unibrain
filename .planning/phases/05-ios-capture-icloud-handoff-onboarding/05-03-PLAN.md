## Phase Goal

**As a** student who recorded on iPhone, **I want to** have my MacBook automatically detect, download, transcribe, and file the recording, **so that** I see a completed note in my course folder without any manual intervention.

## Acceptance Criteria

- [ ] iPhone audio file appears in macOS _inbox/ via iCloud Drive and is detected by NSMetadataQuery (TRIG-01)
- [ ] .icloud placeholder triggers URL.startDownloadingUbiquitousItem and polls until downloaded (IC-04)
- [ ] Files process one at a time FIFO through the full pipeline (TRIG-02)
- [ ] After pipeline success, audio moves from _inbox/ to {vault}/{term}/{course}/ (TRIG-03)
- [ ] Pipeline failure retries 3 times (30s, 2min, 10min), then dead-letters to _inbox/_failed/ (TRIG-04)
- [ ] macOS popover shows "iCloud Inbox: N pending" and queue processing states

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| iCloud Drive _inbox/ -> macOS filesystem | External files arrive via iCloud sync; the pipeline must treat them as untrusted until processed |
| _inbox/_failed/ dead-letter -> User diagnostics | Error sidecar JSON contains metadata; must not leak transcript or audio content |
| NSMetadataQuery results -> Queue processor | Query results are filesystem URLs; must validate they are within the _inbox/ scope |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-09 | Tampering | InboxQueue filename handling | medium | mitigate | IC-03 filenames are system-generated (source-timestamp-uuid pattern); no user-supplied filenames reach the queue; path components validated against traversal |
| T-05-10 | Information Disclosure | DeadLetterHandler error sidecar | medium | mitigate | Sidecar JSON contains only error metadata (timestamp, error type, message, retry count) — never transcript text, audio content, or course names |
| T-05-11 | Denial of Service | InboxQueue + retry storm | low | mitigate | Max 3 retries per file with exponential backoff (30s, 2min, 10min); dead-letter after exhaustion prevents infinite retry loops; queue processes one file at a time (TRIG-02) |
| T-05-12 | Tampering | PipelineWiring.processInboxFile | low | accept | Pipeline inputs are constructed from the filename timestamp (parsed, fallback to file creation date); orchestrator is the existing Phase 3/4 code already handling untrusted transcript data |
| T-05-SC | Tampering | No package installs in this plan | high | mitigate | N/A — zero external packages; all Foundation + UnibrainCore/UnibrainProviders code |
</threat_model>
