<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Keychain (APIKeyStore) | API keys cross from user input → secure storage. Never logged or exposed. |
| .unibrain/consent.json (ConsentStore) | User consent decisions cross from UI → persistent file synced via iCloud. |
| FrontmatterSchema v2 | Provider audit trail crosses from pipeline → note file. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-01 | Tampering | APIKeyStore | critical | mitigate | Use kSecAttrAccessibleWhenUnlocked (never-unlocked). Validate OSStatus on all SecItem calls. Throw on failure (no silent swallow). Unit test verifies security attribute. |
| T-06-02 | Tampering | ConsentStore | medium | mitigate | Atomic writes (.atomic option) prevent iCloud sync corruption. Schema version field enables future migration. Actor isolation prevents concurrent write corruption. Unit test verifies round-trip integrity. |
| T-06-03 | Information Disclosure | ProviderError enum | low | accept | Error cases may expose provider names in logs. This is acceptable — providers are not secrets. API keys NEVER in error strings (verified via grep). |
| T-06-04 | Denial of Service | ModelLoadGate | medium | mitigate | Actor isolation serializes access (Swift 6 guarantee). deny-on-conflict prevents RAM exhaustion (SUMM-07). Unit test covers conflict detection. |
| T-06-05 | Tampering | FrontmatterSchema v2 | low | mitigate | Additive-only change (optional fields) ensures backward compatibility. Decoder treats missing fields as nil (no crash on v1 notes). Unit test covers v1→v2 migration. |
| T-06-06 | Spoofing | MockAPIKeyStore (Linux tests) | low | accept | Mock is in-memory dict for Linux CI only. Production code uses real Keychain. Clear #if os(macOS) guards prevent mock from running on macOS. |
</threat_model>
