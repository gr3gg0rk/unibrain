<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Ollama HTTP (localhost:11434) | Local HTTP requests to Ollama process. No TLS (localhost only). |
| Settings UI → OllamaHealthCheck | User-triggered health checks from Settings. |
| Ollama download (ollama.com) | External site opened via browser. No in-app installer. |
| Summary injection into notes | Generated summary text written to Markdown file. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-07 | Spoofing | Ollama HTTP localhost | low | accept | Localhost-only HTTP. No TLS acceptable for local process. If attacker hosts fake Ollama on localhost:11434, they already have machine access. |
| T-06-08 | Tampering | Summary injection | medium | mitigate | SummarySectionEditor validates HTML markers boundaries before replace. Yams encoding prevents frontmatter corruption. Unit test verifies replace-only-between-markers. |
| T-06-09 | Information Disclosure | Ollama download URL | low | accept | User clicks link to ollama.com (official site). No in-app binary download. Zero risk. |
| T-06-10 | Denial of Service | ModelLoadGate conflict | medium | mitigate | Actor isolation serializes access. SUMM-07 enforced via deny-on-conflict (Task 2 test). No concurrent ASR+LLM load. |
| T-06-11 | Tampering | Regenerate Summary marker replacement | low | mitigate | HTML markers versioned (v=1). Replace only between markers. If markers corrupted, operation fails safely (no blind string replace). Unit test covers marker detection. |
</threat_model>
