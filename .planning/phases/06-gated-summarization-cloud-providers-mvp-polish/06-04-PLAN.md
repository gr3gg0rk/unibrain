## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ConsentSheet → ConsentViewModel | User consent decision crosses from UI to persistent file. |
| CloudFailureSheet → FailureRecoveryViewModel | User retry/fallback decision crosses from UI to provider retry logic. |
| ProviderRouter → Provider clients | Provider selection crosses from Settings to HTTP calls. |
| MenuBarPopover → Sheets | Sheet presentation triggered by pipeline errors. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-18 | Tampering | ConsentViewModel bypass | medium | mitigate | ProviderRouter checks ConsentViewModel BEFORE every provider call (Task 5). Unit test verifies throw on missing consent. No bypass path. |
| T-06-19 | Tampering | ConsentStore corruption | medium | mitigate | Atomic writes (.atomic option) prevent iCloud sync corruption (per 06-01-PLAN.md Task 3). Actor isolation prevents concurrent write corruption. |
| T-06-20 | Information Disclosure | Error message leakage | low | accept | Error messages may expose provider names (acceptable). API keys NEVER in errors (verified via grep). Transcript content never in errors. |
| T-06-21 | Denial of Service | Retry loop without user consent | low | mitigate | CloudFailureSheet requires explicit user action for retry. No automatic retry loops without UI. canRetry() logic prevents retries on unreachable (CF-02). |
| T-06-22 | Tampering | Fallback provider spoofing | low | mitigate | fallbackProvider() returns hardcoded enums (.ollama, .whisperCpp). No user input. No spoofing vector. |
| T-06-23 | Tampering | Sheet presentation bypass | low | accept | Sheets are UI only. Bypassing consent sheet = no cloud call (provider throws .consentDenied). Safe failure mode. |
</threat_model>
