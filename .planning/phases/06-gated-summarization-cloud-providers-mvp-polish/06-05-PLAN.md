<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Settings UI → APIKeyStore | API keys entered in UI cross to Keychain storage. |
| Settings UI → CourseMappingStore | Course mappings changed in UI persist to .unibrain/courses.json. |
| Settings UI → ConsentStore | Consent revocation in Audit tab crosses to consent file. |
| MenuBarPopover → SettingsScene | User-triggered Settings opening crosses from popover to window. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-24 | Tampering | APIKeyEntryRow spoofing | low | mitigate | SecureField masks input (dots). Validation regex checks format before storing. Keychain encrypts at rest. Unit test verifies validation. |
| T-06-25 | Tampering | Settings file corruption | low | mitigate | SettingsViewModel stores state in-memory only. Persistence via ConsentStore/CourseMappingStore (atomic writes). No direct Settings file to corrupt. |
| T-06-26 | Information Disclosure | Vault path exposure | low | accept | Vault path is user-selected folder (not secret). Displaying it in Settings is intentional per 06-UI-SPEC.md. |
| T-06-27 | Tampering | Course mapping injection | medium | mitigate | CourseMappingStore validates JSON schema. Atomic writes prevent corruption. Unit test covers validation. |
| T-06-28 | Denial of Service | Settings window hang | low | accept | Settings UI is responsive by default (SwiftUI). No blocking operations on main thread. Ollama health check runs in background. |
</threat_model>
